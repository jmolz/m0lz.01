# Updated Then Unpublished

Body paragraph.

The most contentious fixture: the post had three update cycles, then
was unpublished. The final frontmatter shows:

- `published: false` (unpublish flip)
- `unpublished_at` set (unpublish finalization timestamp)
- `updated_at` preserved from the LAST update cycle
- `update_count: 3` preserved (update history is not erased by
  unpublish)

Parsers must accept this combination. The lifecycle events are
additive, not exclusive.
