## Overview

m0lz.01 is code-complete (7 phases, 723 tests on 56 suites, all green on `main`) but
is not yet releasable as a public open-source tool. This plan lands everything v0.1.0
requires that the implementation phases did not cover: a shared path-resolution helper
that eliminates a latent **release blocker** (two CLI entry points that resolve
templates against `process.cwd()`), packaging (`npx` installability), GitHub surface
(description, topics, homepage), CI, CHANGELOG + release runbook, and the minimum
README edits needed for a correct install path.

Cutting v0.1.0 itself is OUT of scope â€” this plan only lands everything v0.1.0 needs.

## User Story

As a technical-content practitioner browsing GitHub, I want to discover m0lz.01
via relevant repository topics, understand its value from the repo card in ten
seconds, install it with `npx`, and follow tagged releases, so that I can adopt
it with confidence.

### Patterns to Follow

**Package-relative path resolution â€” new single source of truth:**

```typescript
// src/core/paths.ts
import { fileURLToPath } from 'node:url';
import { existsSync } from 'node:fs';
import { dirname, resolve } from 'node:path';

export function findPackageRoot(moduleUrl: string): string {
  let dir = dirname(fileURLToPath(moduleUrl));
  while (dir !== dirname(dir)) {
    if (existsSync(resolve(dir, 'package.json'))) return dir;
    dir = dirname(dir);
  }
  throw new Error(`package.json not found walking up from ${moduleUrl}`);
}

export const PACKAGE_ROOT = findPackageRoot(import.meta.url);
export const TEMPLATES_ROOT = resolve(PACKAGE_ROOT, 'templates');
```

Walk-up-until-package.json is robust to future file moves and works identically whether the module runs from `src/` (under Vitest) or `dist/` (after compile). No more per-file offset arithmetic.

**Consumer call site (after refactor):**

```typescript
import { TEMPLATES_ROOT } from '../core/paths.js';
const METHODOLOGY_TEMPLATE = resolve(TEMPLATES_ROOT, 'benchmark/methodology.md');
```

**Imports (unchanged from existing convention):**

```typescript
import { fileURLToPath } from 'node:url';
import { resolve, dirname } from 'node:path';
```

- **Acceptance tiered as "in-repo" + "post-merge"**. Executor has a clean stop condition locally; the operator owns post-merge steps. Closes the verification-loop gap both reviewers flagged.
- **PR template links to CLAUDE.md instead of duplicating**. Prevents rule drift between the two files.
